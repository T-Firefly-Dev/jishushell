# JishuShell更新日志 | v0.5.22

---

JishuShell v0.5.22 正式发布。本次更新是一次专门面向稳定性的版本——我们花了一整轮迭代去回收过去几周收到的反馈，集中修复了 **应用安装、Gateway 代理、Provider 接入** 三条主线上的问题。如果你最近遇到过 **白屏、502、Chat 偶发断连、镜像被自动清掉** 等问题，这一版基本都会一并解决。

---

## 稳定性专项：应用安装链路

v0.5.15 引入 Browserless、FileBrowser 等周边应用之后，我们陆续收到一些「**安装看起来成功了，但启动起来不对**」的反馈。这类问题最让人糟心——你分不清是网络、是环境，还是工具本身的问题。这次我们沿着安装链路从头到尾做了一遍体检。

### 预检 zstd 与 Ollama 校验

在调用应用自带的 `install.sh` 之前会先 **预检 zstd**，缺了就先装，避免后续解压静默失败；Ollama 安装完成后会主动做一次 **可执行性校验**，并补齐 `/usr/local/bin/ollama` 软链——之前在某些非规范路径下，Agent 调不起 Ollama 就是因为找不到这个符号。

### Panel 沙箱写入 /usr/local

修复了 Panel 沙箱对 `/usr/local` 不可写导致 **应用安装中途失败** 的问题。

### macOS 安装路径

macOS 上 TypeScript 安装路径补齐了 **launchd 加固脚本**，与 Node 安装路径行为对齐。

### Nomad 镜像 GC

Nomad 的 Docker 镜像 GC 改为关闭，断网或掉电重启之后再也不会出现 **「OpenClaw 镜像被清掉、容器拉不起来」** 的情况。

## 稳定性专项：Gateway 与应用代理

JishuShell 内部有一层 `/apps/:slug/*` 反向代理，把已安装应用的 Web UI 和能力端点接入面板。这一层非常关键，也是最容易出 **「看起来好像没坏，但偶尔出错」** 问题的地方。这次集中清理了一批。

### FileBrowser 白屏与 Token 丢失

**FileBrowser 实例导入后白屏**、Gateway Token 丢失导致 Chat 进不去——已修复。

### 重启后 Chat 状态

服务重启过程中 Chat 偶发 **1006 错误闪烁**、重启后 Chat 仍指向旧 Gateway 链接需要手动刷新——已修复。重启会自动重置 readiness 状态并重新拉取 Launch URL。

### 应用代理 stale host

应用代理偶发路由到 **已停止的旧容器**——根因是注册表的 host 缓存 stale。访问 Capability Endpoint 时会自动修复，整套缓存路径已重写为每次 fresh resolve。

### Capability 代理 Auth Header

Capability 代理 **误把 Panel 自己的 Auth Header 透传给上游**，上游收到陌生 Token 直接 401——这次彻底剥离。

### 自定义运行时端口重分配

自定义运行时端口冲突时 **不会自动重分配**，导致实例反复启动失败——已修复，会自动寻找可用端口。

### 大应用删除超时

体积较大的已装应用删除时接口超时，改为 **后台任务处理**，删除大应用再也不会 502。失败状态的应用恢复后状态显示不准也一并修了——恢复前先做一次健康校验，状态条不再「假绿」。

### 大镜像下载中断

大体积镜像下载因空闲超时被中断，已将 `idle_timeout_ms` 透传到下载步骤，**慢网环境也能拉完**。

## 稳定性专项：Provider 接入与日常体验

### MiniMax VLM 路由

MiniMax 视觉模型路由失败——根因是 Vision 能力推断不准，被当成纯文本模型转发。这次改为 **按 Provider 维度推断能力**，MiniMax 的 VLM 模型现在可以正常被识别和调用。

### 本地 / 私网 Provider

本地 / 私网 Provider 被 SSRF 防护误拦截——包括 **本地 Ollama、vLLM、SGLang** 在内，只要 baseUrl 指向私网地址，现在会自动开启 `allowPrivateNetwork`，不再需要你手动改配置。

### API Key 校验

API Key 校验改为 **真实 completion 探活**。以前只要 Key 格式对了就提示「配置成功」，但真正发请求才发现 Key 早就失效——这次改成保存时直接打一次最小 completion，**无效 Key 当场告诉你**。

### Dashboard 提速

Dashboard 实例列表加载提速——Capability 扫描结果加了缓存，Docker 统计改为批量查询。在 CIX Board 实测 **冷启动从约 1.9 秒降到约 0.87 秒**，实例多的时候体感非常明显。

## 升级方式

```
npm install -g jishushell@0.5.22
```

或通过 Dashboard 顶部的版本更新横幅一键升级。升级完成后建议执行一次 `jishushell restart`，让所有修复的缓存路径都从新进程开始。
