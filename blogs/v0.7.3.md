# JishuShell更新日志 | v0.7.3

---

JishuShell v0.7.3 正式发布。本次更新的主线，是让 **AI Agent 真正管得了你的照片**——我们深度接入了自托管相册 **Immich**：你可以用大白话让 Agent 帮你「找出去年在杭州拍的、有狗的照片」「把这些人的合影归到一个相册」「清一清重复照片」，而所有会动到数据的操作，都会先在一个**审阅画廊**里让你看清楚、点确认后再执行。除此之外，内置知识库的接入方式升级为更稳的**原生 MCP**，联网搜索默认切换到**国内可达的搜索引擎**开箱即用，并修复了 OpenClaw 自更新、加快了仪表盘的状态读取。

---

## 重点更新：AI 照片管家 —— Immich 照片库接入

照片是家庭与个人数据里最「重」也最难整理的一类。这一轮我们把自托管相册 **Immich** 做成了内置应用，并把它和 Agent 打通，让「整理照片」这件事第一次可以用对话完成。

### 一款内置的自托管相册

Immich 现在是可一键部署的内置应用——照片、视频的备份与浏览都在你自己的设备上完成，数据不出门。它由相册服务、机器学习、数据库、缓存等多个容器组成，内部通过应用专属的 Docker 桥接网络互相发现、协同工作，你只需要在应用市场里点一下安装。

<!-- 截图待补：./v0.7.3/immich-app.png —— Immich 应用安装 / 相册主界面 -->

### 让 Agent 帮你「找」和「整理」

打通 Immich 之后，Agent 获得了一整套照片库能力，常用的包括：

- **智能搜图（自然语言 / 以图搜图）**：直接用「草地上玩耍的狗」「海边日落」这类描述检索，背后走 Immich 的视觉智能搜索（CLIP）；也可以「找和这张相似的照片」。
- **按结构化条件筛选**：按拍摄时间、城市 / 省份 / 国家、相机品牌型号、人物、标签、是否收藏 / 归档等精确条件圈定照片。
- **人物、相册、标签的整理**：搜索人物、查看与新建 / 更新相册和标签，把散落的照片归置成清晰的集合。
- **找重复、看分布**：一键找出重复照片、在地图上查看拍摄地分布，帮你腾空间、回顾足迹。
- **网页上传**：需要补充素材时，可直接从网页把照片上传进库，省去中间转存。

### 先审阅、再执行：会动数据的操作有确认关卡

整理照片最怕「一句话说错、批量删错」。为此，凡是会改动数据的动作，Agent 不会直接执行，而是先生成一个**审阅画廊**——把它准备处理的照片可视化列给你看，你确认无误后再一键执行；删相册、删标签这类操作也必须由你在对话里**显式确认**。

同时，删除相册 / 标签只会解除「分组关系」，**不会删除照片本体**，照片仍安全地留在库里。

> ⚠️ 照片涉及个人隐私，且部分操作（删除相册 / 标签）不是随手可撤销的。建议在动手整理前，先通过审阅画廊确认范围，重要相册可先做一次备份。

## 重点更新：知识库更稳、联网搜索开箱即用

### 知识库接入升级为「原生 MCP」

内置知识库 **JishuShell KB** 与 Agent 的对接方式，这一轮升级为 **OpenClaw 原生 MCP**：知识库的检索、上传、从网址导入、文档与索引管理、任务查询等能力，以一套原生工具的形式提供给 Agent，并走远程 MCP 鉴权。相比此前的适配层，接入链路更短、更稳，Agent 调用知识库时的可靠性明显提升。

### 联网搜索默认用国内可达的引擎

内置的 **SearXNG** 联网搜索，默认引擎调整为**国内网络可直接访问**的搜索源（如 Bing、百度等），开箱即可用于联网问答与资料检索，不必再自己折腾引擎配置或代理。

### OpenClaw 自更新修复

修复了 OpenClaw 自更新的两个问题：更新现在会**真正生效并应用**，且界面 / 命令行上报的**版本号如实反映当前实际版本**，不再出现「提示更新了、实际没变」或版本号对不上的情况。

## 更多改进

- **仪表盘更快**：优化了应用与实例状态的读取，仪表盘列表在设备较多、冷启动时的响应更快。
- **多容器应用更可靠**：运行时支持应用专属的 Docker 桥接网络，像 Immich 这类由多个容器组成的应用，服务间可以稳定地按名字互相发现。
- **环境自愈更干净**：`jishushell doctor` 会清理残留的应用 Docker 网络，避免旧网络堆积导致的启动异常。
- **升级 / 修复更稳**：新增目录属主的修复契约，并让启动阶段对代理健康检查的短暂延迟更宽容，减少「刚启动被误判为失败」的情况。
- **知识库候选更清晰**：Agent 的知识库槽位不再把 WeKnora 列为候选，避免误选——内置知识库以 JishuShell KB 为准。
- **安全更新**：处理了一批依赖项安全告警。

**升级方式：**

```bash
npm install -g jishushell@0.7.3
```

或通过 Dashboard 顶部的版本更新横幅一键升级。

如果你是**系统级安装**（已注册 systemd / launchd 服务），`npm install` 只会刷新包文件与 CLI / 面板启动脚本，并不会重启服务、迁移实例。这一步完成后，命令行会用醒目的 `ACTION REQUIRED` 提示你「本地修复尚未执行」，并给出收尾命令——照着复制执行即可。

按提示执行 `sudo jishushell repair`（若 sudo 找不到命令，用提示里的绝对路径 `sudo ~/.jishushell/bin/jishushell repair`）即可完成收尾。`repair` 不只是替换文件，它会先**自检本地状态**（服务是否需重启、运行时配置、provider 密钥兼容性、运行中实例是否需重启），再统一**对账并重载系统组件**：重新装好 core / 面板的 systemd 服务、重启并验证 core（:8091）与面板（:8090），最后给出「System reload complete」。整个过程一条命令搞定，把「升级后跑不起来」的概率压到最低。

收尾完成后，刷新任意已打开的 JishuShell 浏览器标签页，即可加载更新后的界面。

> 普通的 `npm install -g jishushell`（非系统级服务安装）不会出现这一步，包文件刷新后即可直接使用。
